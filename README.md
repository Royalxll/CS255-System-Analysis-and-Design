# CS255-System-Analysis-and-Design
Project Summary:
DrivePass wanted a system that can help students get ready for their driving test through online practice exams and driving lessons. And the system will have features such as registration, scheduling, training management, and administrative functions.
What I did well:
I feel that I was able to effectively gather and document what the client needed and turn that into UML diagrams and system designs that will align with their needs.
Revision and Improvements:
If I could make revisions, I would improve the user interface mockups and add more details to the system workflows and design documentation.
Interpreting User Needs:
I was able to use the interview transcript and requirements to design the system features that will meet the client's goals. Keeping the user needs in mind is very important, that is because it makes sure the system is useful, efficient, and user-friendly.
Software Design Approach:
I started with analyzing the requirements and stakeholder needs, then using the UML diagrams, process modeling, and documentation to help me making the design. And for future reference I will continue using these techniques to make effective and reliable systems.
